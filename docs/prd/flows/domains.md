> **单一出现原则**:证书状态机定义在 flows/certificates.md;本文件引用其状态,不复述定义与流转。域名不定义自身证书态。

## 1. 模块职责与边界

本模块管理**"域名"这一对象本身**:维护域名列表(增删改查)、域名的备注与分组、域名与验证方式的关联,以及从域名侧查看并进入其证书关联。域名是 autohttps 管理的核心对象(glossary),证书围绕域名而签发。

### 1.1 本模块负责

- 维护域名对象:hostname、是否通配符(由 hostname 判定)、分组、备注。
- 维护域名与**验证方式**的关联(该域名倾向用哪种验证方式类别);方式的配置与执行明细委托 acme。
- 从域名侧呈现其**证书态投影**(该域名当前被哪些证书覆盖、各证书状态),并提供进入 certificates 发起签发 / 查看证书的入口。

### 1.2 委托与消费(边界)

| 事项 | 归属模块 | 本模块的关系 |
| --- | --- | --- |
| 证书生命周期(签发 / 续签 / 吊销 / 删除 / 导出、证书状态机) | `certificates` | 本模块**提供**域名对象供证书关联;从域名侧**消费 / 反查**证书态(投影),不持有证书生命周期、不定义证书状态 |
| 验证方式明细(HTTP-01 webroot / DNS-01 TXT 的配置与挑战执行、ACME 账户) | `acme` | 本模块只维护"域名 ↔ 验证方式"的**关联(类别选择)**;方式的具体配置与执行**委托** acme |
| 域名 ↔ 证书的关联链(证书的 SAN 域名清单) | `certificates` | 关联链记录在**证书侧**(证书列出其覆盖域名);本模块从域名侧**反向查看**,关联的建立发生在 certificates 签发流程 |
| 红点 / 到期高亮 / 状态总览 | `dashboard` | 本模块**提供**域名与其证书态数据;提醒呈现归 dashboard |

### 1.3 明确不做

- ❌ 证书签发 / 续签 / 吊销 / 导出等生命周期动作(归 certificates;本模块仅提供入口)。
- ❌ 验证方式的配置与挑战执行(归 acme)。
- ❌ 批量操作(批量新增 / 批量删除域名)、域名维度的统计卡片(总览归 dashboard)。
- ❌ 域名启用 / 停用 / 归档等独立生命周期(需求未明示,见 §2 与 §5-DECD1)。

## 2. 状态说明:域名无独立生命周期状态机

域名**不设独立的生命周期状态机**。理由:域名对象一经新增即"存在",其后有业务含义的变化只有两类——要么是**元数据编辑**(分组 / 备注 / 验证方式关联,无状态语义),要么是**证书态**,而证书态完全派生自 certificates 的证书状态机。硬造域名状态机只会与证书态重叠、制造漂移点。

### 2.1 证书态投影(引用不复述)

域名列表 / 详情所呈现的各类证书态(有效 / 即将到期 / 失败 / 已过期 等),均为 **certificates 证书状态机(flows/certificates.md §2)在域名维度的投影**,本文件引用不复述其状态定义与流转。域名维度另有一个**非证书状态**的呈现条件,归本模块:

| 域名呈现条件 | 含义 | 来源 |
| --- | --- | --- |
| 无证书 | 该域名尚未被任何证书关联(新增后未签发,或原关联证书已全部删除) | 本模块(关联的有无) |
| (各证书态) | 该域名被至少一张证书关联时,取其关联证书的状态呈现 | 投影自 certificates |

> 一个域名可被多张证书关联(见 §4),列表呈现时的聚合口径(如取"最紧急"的关联证书态)在页面 PRD 展开;证书状态定义本身不在此复述。

## 3. 主业务流程

> 只描述业务动作序列与模块委托,不含界面细节(界面在页面 PRD)。

### 3.1 新增域名

1. 使用者录入 hostname;系统校验格式,并据 `*.` 前缀判定是否为**通配符域名**(glossary)。
2. 校验同一实例内 hostname **不重复**。
3. 可选:设置分组、备注、关联验证方式(见 §3.5)。
4. 保存 → 域名对象建立;此时其证书态为"无证书"。

### 3.2 编辑域名

1. 使用者修改域名的**分组 / 备注 / 验证方式关联**。
2. hostname **不可编辑**(它是域名的身份);需改名等同于"删除旧域名 + 新增新域名"(见 §5-DECD2)。

### 3.3 删除域名

1. 使用者对某域名发起删除,二次确认。
2. **前置校验**:若该域名仍被**任一现存证书**关联,则**拦截删除**,提示"该域名仍被 N 张证书关联,请先在证书管理处理相关证书"。
   - 依据 certificates 不变量:"证书必须关联已存在的域名"(flows/certificates.md §1.2 / modules/certificates.md DS2);删除被关联域名会使证书出现悬空引用。
   - 本模块不代为删除 / 改动证书(边界),须由使用者在 certificates 处理后再删域名。
3. 未被任何证书关联("无证书")时:移除该域名对象及其元数据。
   - 说明:删除**证书**不删其域名(域名是独立对象,可无证书而存在);删除**域名**需其先无证书关联——两侧删除不互相级联。

### 3.4 关联证书(查看 + 签发入口)

1. 使用者在域名详情查看该域名被哪些证书关联及各证书状态(投影自 certificates)。
2. 提供入口跳转 certificates:为该域名**发起签发**(预置本域名),或查看某关联证书详情。
3. 说明:签发动作与"域名 ↔ 证书关联"的**建立**均在 certificates 完成(flows/certificates.md §3.1);本模块只提供域名侧的查看与入口。

### 3.5 关联验证方式

1. 使用者为域名设置 / 查看其**验证方式关联**(类别:HTTP-01 / DNS-01)。
2. **约束**:通配符域名只能关联 DNS-01(glossary:通配符仅经 DNS-01 签发)。
3. 说明:此处仅建立"域名 ↔ 验证方式类别"的关联(选择);验证方式的具体配置(webroot 路径 / DNS TXT 等)与挑战执行**委托 acme**,签发时由 certificates 取用本关联并交 acme 执行。

## 4. 与 certificates 的关联口径

| 维度 | 口径 |
| --- | --- |
| 关联基数 | **域名 ↔ 证书 多对多**:一张证书可关联多个域名(SAN,至多一个通配符);一个域名可被多张证书关联(如新旧证书并存、不同证书覆盖) |
| 关联链归属 | 记录在**证书侧**(证书列出其 SAN 域名);domains 从域名侧反向查看 |
| 关联建立时机 | 在 certificates **签发流程**中选定域名时建立(flows/certificates.md §3.1);domains 不产生该关联 |
| 证书态来源 | 域名呈现的证书态**投影自** certificates 证书状态机;domains 不定义、不复述 |
| 域名前置约束(供 certificates 消费) | 证书关联的域名必须已在 domains 存在;通配符域名仅经 DNS-01 |

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 modules/domains.md §5 互补(此处记流程 / 边界相关决策)。

- **DECD1(2026-07-16)· 域名不设独立生命周期状态机**:域名有业务含义的变化仅为元数据编辑或证书态投影,不引入"启用 / 停用 / 归档"等状态。
  - 为什么不做:需求(project §5)未明示域名启停语义;硬造状态机会与 certificates 证书态重叠并制造漂移点。若后续出现"停用域名 = 暂停其自动续签"等真实诉求,再增量定义。
- **DECD2(2026-07-16)· hostname 不可编辑,改名 = 删 + 增**:编辑仅限分组 / 备注 / 验证方式关联;hostname 为域名身份,不可变更。
  - 为什么:hostname 变更会使已关联证书的 SAN 与域名对象错位;以"删旧增新"替代可避免悬空引用,且符合"主标识不可变"的通行建模。
- **DECD3(2026-07-16)· 删除域名以"是否被证书关联"硬拦截**:被任一现存证书关联的域名不可删除,须先在 certificates 处理相关证书。
  - 为什么不做级联删除:级联删除域名会反向删动证书,越过 certificates 边界且风险高;硬拦截保持不变量("证书必关联存在域名")且让删除动作留在各自模块。**已裁决(orchestrator, 2026-07-16):删除域名硬拦截,被任一现存证书关联即不可删(须先在 certificates 处理相关证书);不对已吊销 / 已过期证书放宽。**
