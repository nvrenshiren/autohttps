## 1. 目的

新增根 CA 页用于**往根 CA 列表里加入一个根 CA**,含两种方式:**创建新根 CA**(本地生成密钥对并自签)与**导入已有根 CA**(提供根 CA 证书 + 配对私钥)。它是根 CA 进入状态机的入口(flows/local-ca.md L1 / L2)。

创建 / 导入均为**本地离线操作**,**瞬时成功 / 失败**——要么立即产出根 CA,要么立即失败,**无「创建中 / 导入中」过渡态**(LC1)。

---

## 2. 功能清单

| # | 功能 | 说明(业务口径) |
| --- | --- | --- |
| F1 选择新增方式 | 二选一:创建新根 CA / 导入已有根 CA |
| F2 创建新根 CA | 录入名称 / 标识、有效期等基本信息;本地生成密钥对并自签,产出新根 CA,进入「有效」(L1) |
| F3 导入已有根 CA | 提供根 CA 证书 + 配对私钥(私钥受口令保护时提供解密口令);校验通过进入「有效」;导入证书本身已过期则判「已过期」(L2) |
| F4 提交 | 创建 / 导入立即成功(根 CA 进入有效 / 已过期)或立即失败(提示原因,不产生条目);无过渡态 |

> 本页**不含**:批量创建 / 批量导入(需求未明示);根 CA 的续期 / 自动重建(LC3);技术参数(密钥算法等)的业务录入(由技术层定);内网证书相关操作(归 certificates)。

---

## 3. 页面流转

| 触发 | 去向 / 结果 | 说明 |
| --- | --- | --- |
| 创建成功 | 新根 CA 进入「有效」→ 通常跳根 CA 列表 / 详情页 | 可立即用于签发与导出 |
| 导入成功 | 进入「有效」(导入证书已过期则「已过期」)→ 跳列表 / 详情页 | 与新建根 CA 等同参与后续流程 |
| 创建 / 导入失败 | 留在本页并提示失败原因 | 参数非法 / 证书私钥不配对 / 口令错 / 非可用根 CA;不产生根 CA 条目 |
| 取消 | 返回根 CA 列表页 | 不新增任何根 CA |

> 进入本页的来源包括:根 CA 列表页「创建 / 导入」入口、根 CA 详情页「已过期」处置引导。

---

## 4. 交互说明

- **两种方式二选一**:创建新根 CA / 导入已有根 CA。
- **创建新根 CA**:录入名称 / 标识、有效期(生效至失效)等基本信息(密钥算法等技术参数由技术层确定,不在业务录入);本地生成密钥对并自签,产出新根 CA;成功后进入「有效」,可立即用于签发内网证书与导出。
- **导入已有根 CA**:提供已有根 CA 证书与其配对私钥;私钥受口令保护时提供**解密口令**;校验证书与私钥**配对**、且证书为**可用根 CA**;通过后进入「有效」;若导入的证书**本身已过有效期**,导入后直接判为「已过期」。
- **本地离线 / 瞬时**:创建与导入均为本地离线操作(无需外网,project §7 离线可用);要么立即成功、要么立即失败(参数非法 / 证书私钥不配对 / 口令错误 / 非可用根 CA),**不产生「创建中 / 导入中」过渡态**(与 certificates 的在线签发过渡态不同,LC1)。
- **失败处理**:失败留在本页提示具体原因,不产生根 CA 条目;修正后可重新提交。
- **敏感数据**:创建生成的私钥、导入提供的私钥均属敏感数据(project §7),其安全存储由技术层保障;本页只负责录入 / 提交。
- **成功去向**:成功后新根 CA 出现在根 CA 列表,可进入其详情导出根 CA 证书供客户端信任。

---

## 5. 验收要点

> 业务口径,每条可判真伪。

1. 可选择两种新增方式:创建新根 CA 或 导入已有根 CA。
2. 创建新根 CA 可录入名称 / 标识、有效期等基本信息。
3. 创建成功后新根 CA 进入「有效」,可立即用于签发与导出。
4. 导入已有根 CA 需提供根 CA 证书与配对私钥;私钥受口令保护时需提供解密口令。
5. 导入时校验证书与私钥配对、且证书为可用根 CA;不配对 / 口令错 / 非可用根 CA 则导入失败并提示原因。
6. 导入的证书本身已过有效期时,导入后直接判为「已过期」。
7. 创建与导入均为本地离线操作,无需外网。
8. 创建 / 导入要么立即成功、要么立即失败,不出现「创建中 / 导入中」过渡态。
9. 创建 / 导入失败时留在本页并提示失败原因,不产生根 CA 条目。
10. 成功后新根 CA 出现在根 CA 列表页。
11. 本页不含批量创建 / 批量导入。
