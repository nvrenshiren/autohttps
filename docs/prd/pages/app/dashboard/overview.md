## 1. 目的

总览页是 autohttps 的**启动首屏**:桌面 / 服务器形态启动后第一眼看到的页面。它把散落在各模块的证书状态**聚合成一屏可读的总览**——多少证书、谁即将到期、谁出了问题——并以红点 / 待处理高亮把「该处理的证书」推到眼前(project §1「启动即检测、看到红点就知道谁该续」)。

本页是**纯聚合视图 + 快速入口**:只呈现与跳转,**不含任何证书 / 域名 / 任务的增删改**;所有处置动作都跳到对口页面完成。

## 2. 功能清单

| # | 功能 | 说明(业务口径) |
| --- | --- | --- |
| F1 概览三指标 | 展示证书总数 / 即将到期数 / 失败数三项指标;口径见 flows/dashboard.md §2 |
| F2 待处理清单 | 汇聚处于「已过期 / 签发失败 / 续签失败 / 即将到期」状态的证书,逐条呈现(关联域名、当前状态、有效期);告警级(已过期 / 签发失败 / 续签失败)优先于关注级(即将到期),已过期居首 |
| F3 红点提醒 | 存在任一待处理证书即亮红点,可显示待处理计数;全部处置完成后自动清零。桌面形态红点同时体现在系统托盘图标角标(仅桌面);服务器形态体现在浏览器内(仅服务器) |
| F4 待处理项跳转 | 从待处理项逐条跳到该证书详情去处理(续签 / 重试 / 吊销 / 删除),或跳到其最近任务查看失败原因 |
| F5 常用操作入口 | 提供跳向「发起签发 / 查看全部证书 / 域名管理 / 任务与历史」的快速入口 |

> 本页**不含**:证书 / 任务 / 域名的任何增删改执行、证书状态的判定与扫描(归 certificates)、多渠道通知、按签发方式 / CA 的分布图表等 project §5 未明示的统计维度(modules/dashboard.md §2 边界)。

## 4. 交互说明

- **启动即呈现**:启动后本页即为首屏,读取 certificates 启动扫描后的最新状态呈现,无需使用者额外操作(flows/dashboard.md §5)。
- **三指标呈现**:证书总数、即将到期数、失败数三项数值一屏可见;「即将到期数」+「失败数」恰等于待处理清单条数(总览与清单口径一致,不出现对不上)。
- **待处理清单排序**:告警级(已过期 / 签发失败 / 续签失败)排在关注级(即将到期)之前,已过期居首(flows/dashboard.md §3.1 优先级);每条显示关联域名、当前状态、有效期。
- **红点**:只要存在任一待处理证书即亮,可显示计数;所有待处理项被处置(续签成功 / 删除 / 吊销成功等)后自动清零。
  - 桌面形态(仅桌面):红点同时体现在系统托盘图标角标与主窗口内;关窗不退出时托盘角标仍提示待办。
  - 服务器形态(仅服务器):无系统托盘,红点体现在浏览器内。
  - 红点的**触发规则两形态一致**,仅呈现载体不同。
- **纯呈现 + 跳转**:本页不提供任何增删改;点待处理项进入对应证书详情或其最近任务,处置在目标页面完成。
- **随状态刷新**:certificates 定期扫描推进证书状态,本页聚合随之刷新(刷新机制由 architect 设计,业务只要求「呈现与证书当前状态一致」)。

## 5. 验收要点

> 业务口径,每条可判真伪。

1. 启动后本页即为首屏并直接展示总览,无需额外操作。
2. 证书总数 = 当前全部**未删除**证书条目数(含各状态,含已吊销);已删除证书不计入。
3. 即将到期数 = 处于「即将到期」状态的证书数。
4. 失败数 = 处于「已过期 / 签发失败 / 续签失败」三类状态的证书数之和。
5. 「即将到期数」+「失败数」= 待处理清单条数(两者始终一致)。
6. 待处理清单只含「已过期 / 签发失败 / 续签失败 / 即将到期」四类状态的证书,不含「有效 / 待签发 / 签发中 / 续签中 / 吊销中 / 已吊销」。
7. 待处理清单中告警级(已过期 / 签发失败 / 续签失败)排在关注级(即将到期)之前;已过期排在最前。
8. 每条待处理项显示关联域名、当前状态、有效期。
9. 存在任一待处理证书时红点亮起;所有待处理项处置完成后红点清零。
10. 桌面形态红点同时体现在系统托盘图标角标(仅桌面);服务器形态红点仅体现在浏览器内(仅服务器)。
11. 点击待处理项进入该证书的证书详情页;点击其失败原因入口进入该证书的最近任务。
12. 快速入口可分别直达:签发证书页 / 证书列表页 / 域名列表页 / 任务与历史。
13. 本页不提供任何证书 / 域名 / 任务的增删改操作。
14. 「吊销失败」不单独触发红点;若吊销失败回退后的状态本身属待处理四类之一,则按该状态正常出红点(flows/dashboard.md §3.2)。
