## 1. 概述

证书管理是 autohttps 证书主线的落点模块:围绕**证书这一核心实体**,提供列表与详情查看、手动签发 / 续签触发、自动续签的执行响应、吊销、删除与证书文件导出。证书的完整生命周期由**证书状态机**(flows/certificates.md)刻画。

本模块**聚焦证书结果与状态流转**:签发 / 续签时把"域名验证"委托 `acme`、把"自签根 CA 签发"委托 `local-ca`,消费 `settings` 的续签策略,经 `tasks` 执行并留痕,把状态数据提供给 `dashboard` 呈现红点。证书交付止于**签发 + 本地存储 + 到期 / 失败提醒**,导出后由使用者自行部署,本模块**不做自动部署**(project §9-D3)。

MVP 单一角色 operator 对本模块全权(roles.md),无细粒度权限差异。

## 2. 功能列表(按端分组)

> 端点只有一个 `app`(桌面形态 / 服务器形态共享前端);形态差异在功能项内以"仅桌面 / 仅服务器"标注。

### 2.1 端 `app`

**A. 查看**

| 功能 | 说明 |
| --- | --- |
| A1 证书列表查看 | 展示全部证书及其当前状态、关联域名、有效期、签发方式(ACME / 自签);支持按状态、签发方式、域名关键字筛选查看 |
| A2 证书详情查看 | 单张证书完整信息:关联域名、当前状态、有效期(生效 / 失效时间)、签发方式与来源、证书链概要、可执行操作入口 |

**B. 签发**

| 功能 | 说明 |
| --- | --- |
| B1 发起首次签发 | 为选定域名创建证书条目并触发首次签发;选择签发方式(公共 ACME / 自签根 CA);验证细节委托 acme,自签委托 local-ca |
| B2 签发失败重试 | 对"签发失败"证书手动重试;失败原因日志见 tasks |

**C. 续签**

| 功能 | 说明 |
| --- | --- |
| C1 手动续签 | 对已签发证书手动触发续签(有效状态允许提前续) |
| C2 自动续签响应 | 对"即将到期"证书按 settings 续签策略自动触发续签(策略配置归 settings,调度 / 任务归 tasks,本模块执行续签并更新状态) |
| C3 续签失败重试 | 对"续签失败"证书查看失败原因(日志见 tasks)并手动 / 自动重试 |

**D. 吊销与删除**

| 功能 | 说明 |
| --- | --- |
| D1 吊销证书 | 对有效 / 即将到期 / 已过期 / 续签失败证书发起吊销;ACME 证书向 CA 吊销,自签证书由 local-ca 根 CA 标记作废 |
| D2 删除证书 | 移除非进行中态证书的条目与本地文件,二次确认;关联未完成任务由 tasks 一并清理 |

**E. 导出**

| 功能 | 说明 |
| --- | --- |
| E1 导出证书文件 | 导出叶子证书 / 私钥 / 证书链供使用者自行部署;私钥属敏感数据需风险提示;仅桌面可保存到本地路径、仅服务器为浏览器下载(形态差异);具体导出项与格式在页面 PRD 展开 |

> 本模块**不含**:批量签发 / 批量吊销 / 批量删除等批量操作、证书数量统计卡片(总数 / 即将到期数统计归 dashboard)、自动部署(project §6.2)。

## 4. 数据来源

> 供 architect 作为唯一设计依据,用**业务语言**描述本模块涉及哪些数据及其来源;**不含**表结构 / 字段类型 / 存储机制 / API(由 architect 设计)。敏感数据的具体存储机制由 architect 依 project §7 数据安全约束确定。

| # | 数据 | 来源 | 说明 |
| --- | --- | --- | --- |
| DS1 | 证书条目(本模块核心数据) | **本模块产生并维护** | 每张证书的:关联域名、签发方式(ACME / 自签)、当前状态(证书状态机)、有效期(生效 / 失效时间)、签发时间、证书标识(如序列号 / 指纹)、本地证书文件与私钥的存储位置引用 |
| DS2 | 关联域名 | `domains` | 证书必须关联一个或多个已存在的域名;可含通配符域名;域名的增删改归 domains |
| DS3 | 签发 / 验证来源信息 | `acme`(公共 CA)/ `local-ca`(自签) | 本模块记录"用哪种方式、经哪个 ACME 账户 / 哪个根 CA 签的";验证方式与挑战结果明细归 acme,根 CA 明细归 local-ca |
| DS4 | 续签策略 | `settings` | 提前天数、自动续签开关;本模块消费以判定"即将到期"与是否自动续签,不定义策略 |
| DS5 | 签发 / 续签 / 吊销任务与执行历史 | `tasks` | 本模块触发任务、依据任务结果更新证书状态;每次执行的时间 / 结果 / 日志 / 重试记录归 tasks |
| DS6 | 证书文件本体(叶子证书 / 私钥 / 证书链) | **本模块产生**(签发 / 续签结果落地) | 签发成功后本地安全存储;私钥属敏感数据(project §7);导出时读取 |

> **关联基数(待 architect 关注)**:一张证书可关联一个或多个域名(SAN),其中至多一个可为通配符域名;通配符域名仅经 DNS-01 签发。此为当前采用的默认口径,若需收敛为"一证一域"见 §5-DEC4。

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 flows/certificates.md §4 互补(此处记模块 / 功能 / 数据相关决策)。

- **DEC1(2026-07-16)· 证书交付止于导出,不自动部署**:承接 project §9-D3;本模块提供证书文件导出,部署由使用者自行完成。
  - 为什么不做部署:目标服务与部署方式差异极大,自动部署复杂度与风险远高于收益。
- **DEC2(2026-07-16)· 不做批量操作与统计卡片**:列表仅提供基础筛选查看,不做批量签发 / 吊销 / 删除;证书数量统计归 dashboard。
  - 为什么:需求未明示批量场景;统计总览是 dashboard 职责,避免跨模块重复。
- **DEC3(2026-07-16)· 页面清单收敛为 3 页**:列表页、详情页、签发证书页;续签 / 吊销 / 删除 / 导出作为列表 / 详情页上的操作(确认框 / 面板),不单列页面。
  - 为什么不拆:这些操作是对已有证书的动作,依附于详情 / 列表上下文;单列页面会割裂操作与对象。
- **DEC4(2026-07-16)· 默认支持一证多域(SAN)**:证书可关联多个域名,含至多一个通配符域名。
  - 为什么不默认"一证一域":一证多域是真实证书与 ACME 的标准能力,采一对多关联更通用,后续收敛为一证一域是其子集、返工小于反向扩展。**已裁决(orchestrator, 2026-07-16):保留一证多域(SAN,至多一个通配符),不收敛为一证一域。**
