## 1. 概述

任务与历史是 autohttps 证书主线的**执行与留痕层**:certificates 触发的"签发 / 续签 / 吊销"在本模块落成一次次可排队、可执行、可重试、可取消的**任务**,并保存每次执行的**时间 / 结果 / 日志 / 重试历史**。任务的完整生命周期由**任务状态机**(flows/tasks.md)刻画。

**任务与证书是两个不同实体**:证书是被作用对象(状态与有效期归 certificates),任务是对某张证书的一次执行单元(排队 / 执行 / 结果归本模块)。本模块依任务执行结果**驱动** certificates 状态流转(签发成功→有效 / 续签失败→续签失败 / 吊销成功→已吊销 …),但**不改写、不复述**证书状态机;把"某证书最近一次任务的结果与时间"提供给 dashboard 排查跳转(dashboard DS3);消费 settings 的续签策略与自动再尝试口径(settings SF2),但**不持有**续签策略、重试参数与到期扫描器。

MVP 单一角色 operator 对本模块全权(roles.md:查看历史 / 重试 / 取消任务),无细粒度权限差异。

## 2. 功能列表(按端分组)

> 端点只有一个 `app`(桌面形态 / 服务器形态共享前端);任务与历史无形态特有能力,功能两形态一致。触发决策、状态流转与联动规则见 flows/tasks.md,此处只列功能、不复述规则。

### 2.1 端 `app`

**A. 查看**

| 功能 | 说明 |
| --- | --- |
| A1 任务列表查看(队列 + 历史) | 统一列表呈现全部任务:任务类型(签发 / 续签 / 吊销)、关联证书(及其域名)、当前任务状态(排队 / 执行中 / 成功 / 失败 / 已取消)、触发方式(手动 / 自动 / 清理)、入队 / 开始 / 结束时间、结果摘要;支持按类型、状态、关联证书、时间范围筛选。进行中(排队 / 执行中)与已结束(成功 / 失败 / 已取消)同表可筛,"队列"即按进行中态筛选的视图 |
| A2 任务详情与执行日志查看 | 单个任务的完整信息:类型、关联证书与当时状态、触发方式、各时间点、执行结果、失败原因、**完整执行日志**(验证步骤 / CA 交互 / 错误详情),以及**重试链**(本次是第几次尝试、前序失败任务与后继重试任务的引用);只读 |

**B. 执行控制**

| 功能 | 说明 |
| --- | --- |
| B1 手动重试失败任务 | 对"失败"任务(签发 / 续签 / 吊销均可)发起重试,**派生一个新任务**入队(同类型、同证书);原失败任务保留于历史,新任务记入重试链。重试驱动证书从告警态回到进行中态(签发失败→签发中 / 续签失败→续签中,或对已回退证书重新发起吊销;见 flows/tasks.md §4) |
| B2 取消任务 | 对"排队"或"执行中"任务发起取消 → 已取消。进行中取消为**尽力而为**(在途 CA 操作可能仍生效,由 certificates 下次扫描校正);证书侧回退 / 落点由 certificates 承接(已裁决:certificates T21–T24,见 flows/certificates.md §2.3 / flows/tasks.md §4.4) |

**C. 队列与调度(系统行为,非交互)**

| 功能 | 说明 |
| --- | --- |
| C1 任务入队与调度执行 | 接收 certificates 触发的签发 / 续签 / 吊销,创建任务入队并按序调度执行;执行中委托 acme 验证、委托 local-ca 自签、或与 CA 交互;记录全过程时间、结果与日志 |
| C2 进程中断恢复 | 进程异常中止后启动时,对遗留"排队 / 执行中"任务做恢复判定(重新排队执行 / 校正为失败可重试),不使任务永久卡死;恢复机制由 architect 定,业务底线是"不卡死、可被 operator 看到并处置"(flows/tasks.md §3.3) |
| C3 自动续签 / 自动再尝试的执行承接 | 承接 certificates 依 settings 自动续签开关(经周期扫描)发起的自动续签、以及对"续签失败且未过期"证书的自动再尝试,均作为新的续签任务执行;**本模块不判定到期、不持有重试次数 / 间隔**(触发决策归 certificates + settings,settings SF2) |
| C4 证书删除清理 | 证书删除时,取消该证书"未完成(排队 / 执行中)"任务(→ 已取消);该证书历史任务记录保留为只读("证书已删除"标注),不级联删除(§5-DEC3) |

> 本模块**不含**:批量重试 / 批量取消等批量操作、跨证书操作审计视图(谁在何时做了什么,roles §4 后置)、证书到期判定与扫描(归 certificates)、续签策略 / 重试参数定义(归 settings)、多渠道通知(仅向 dashboard 提供数据,project §6.2)、手动编辑任务参数 / 手动新建任务(任务只由 certificates 触发或对失败任务重试派生)。

## 4. 数据来源

> 供 architect 作为唯一设计依据,用**业务语言**描述本模块涉及哪些数据及其来源;**不含**表结构 / 字段类型 / 存储机制 / API(由 architect 设计)。敏感数据的具体存储机制由 architect 依 project §7 数据安全约束确定。

| # | 数据 | 来源 | 说明 |
| --- | --- | --- | --- |
| DS1 | 任务记录(本模块核心数据) | **本模块产生并维护** | 每个任务的:任务类型(签发 / 续签 / 吊销)、关联证书引用、触发方式(手动 / 自动 / 清理)、当前任务状态(任务状态机)、入队时间、开始执行时间、结束时间、执行结果(成功 / 失败 / 已取消)、失败原因摘要、重试关系(尝试序号 / 前序失败任务引用 / 后继重试任务引用) |
| DS2 | 执行日志 | **本模块产生** | 每次执行的过程日志:验证 / 挑战步骤、与 CA 的交互、错误详情;供任务详情查看与失败排查。**私钥 / 账户密钥等敏感数据不得进入日志**(project §7)—— architect 注意脱敏 |
| DS3 | 关联证书 | `certificates` | 任务作用的证书对象引用及其当时状态;任务据结果驱动证书状态流转。证书本体、状态与有效期归 certificates,本模块引用不复述;证书被删除后,历史任务保留对其的只读引用(标注"证书已删除",见 §5-DEC3) |
| DS4 | 触发与委托执行来源 | `certificates`(触发)/ `acme`(验证挑战)/ `local-ca`(自签签发 / 吊销标记) | 任务由 certificates 触发;执行到"域名验证"环节由 acme 承接、"自签签发 / 吊销标记"由 local-ca 承接。本模块记录"任务经哪种方式执行"及其结果引用,验证明细归 acme、根 CA 明细归 local-ca,本模块不复述 |
| DS5 | 续签策略与自动再尝试口径 | `settings` | 本模块消费 certificates 依 settings(续签提前天数 / 自动续签开关)发起的触发结果;自动再尝试依附自动续签开关下的周期扫描(settings SF2:**无独立重试次数 / 间隔参数**)。本模块不定义策略、不存储策略值 |
| DS6 | 数据存储根路径 | `settings` | 任务历史与执行日志落于 settings 定义的存储根路径下(settings §4);本模块只在此根路径下持久化,不定义路径本身 |

> **数据量与保留(待 architect 关注)**:一张证书随时间累积多条任务(首签 + 多次续签 + 失败重试 + 吊销),任务历史为**只增**数据;MVP 不做自动清理 / 归档 / 保留期上限(需求未明示),历史长期保留(含已删除证书的只读任务,§5-DEC3)。列表页需具备应对历史增长的筛选 / 分页能力(交互细节在页面 PRD)。任务并发执行上限、调度顺序、到期扫描周期均属实现机制,由 architect 设计;业务要求仅为"任务终态与证书状态一致、进行中任务不永久卡死"。

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 flows/tasks.md §6 互补(此处记模块 / 功能 / 数据 / 页面相关决策)。

- **DEC1(2026-07-16)· 页面收敛为 2 页(列表页 + 详情页)**:队列与历史合于一个列表页(同一批任务按状态筛选),任务详情页承载执行日志与重试链;重试 / 取消作为列表 / 详情上的动作(确认框),不单列页面。
  - 为什么不拆:队列与历史是同一批任务的不同筛选视角,拆两页会制造"同一数据两处维护"的漂移;重试 / 取消是对具体任务的动作,依附任务上下文,单列页面会割裂动作与对象(对齐 certificates DEC3 精神)。
- **DEC2(2026-07-16)· 不做批量操作与跨证书操作审计**:列表仅提供基础筛选与逐任务的重试 / 取消,不做批量重试 / 批量取消;"历史"聚焦签发 / 续签 / 吊销的执行留痕,不做"谁在何时做了什么"的操作审计视图。
  - 为什么:需求未明示批量场景;操作审计(含多用户归因)属 roles §4 后置项,与 MVP 无鉴权、单角色前提不匹配,避免主动扩张功能(对齐 certificates DEC2 / dashboard DD3)。
- **DEC3(2026-07-16)· 证书删除后任务历史保留为只读,不级联删除**:删除证书取消其未完成任务;历史任务记录以只读("证书已删除"标注)长期保留,不随证书一并删除。
  - 为什么不级联:tasks 模块职责即"执行历史留痕"(project §5),级联删除会丢失"某证书曾如何签发 / 失败"的历史;只读保留避免对已删证书误触发重试。**已裁决(orchestrator, 2026-07-16):保留为只读,不级联删除。**
- **DEC4(2026-07-16)· 任务与证书为不同实体,任务不复述证书状态**:任务是"一次执行单元",证书是"被作用对象";任务:证书 = 多:一(一证多次执行);任务据结果驱动证书状态流转,但证书状态定义与真相唯一归 certificates。
  - 为什么:orchestrator 明确二者为不同实体;一证多次执行天然多任务;复述证书状态会制造双真相、违反单一出现原则,故本模块只引用证书状态编号、不复述。
- **DEC5(2026-07-16)· 任务只由 certificates 触发或对失败任务重试派生,不支持手动新建 / 编辑任务**:本模块不提供"手动创建一个任意任务"或"编辑任务参数"的入口。
  - 为什么:任务是证书生命周期动作的执行落点,其类型 / 目标证书 / 参数由证书侧动作(签发 / 续签 / 吊销)决定;允许裸建 / 改参会绕过证书状态机、制造与证书不一致的孤儿执行,违背"任务服务于证书生命周期"的定位。
