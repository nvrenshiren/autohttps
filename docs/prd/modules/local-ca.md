## 1. 概述

自签根 CA 是 autohttps 面向**内网 / 本地开发场景**的统一签发能力:让使用者创建或导入一个自己的根 CA,把它导出并安装到客户端信任库后,即可用它签发被信任的内网证书,解决"根 CA 管理混乱"这一痛点(project §3)。

本模块**只负责根 CA 本身**:创建 / 导入 / 查看 / 导出根 CA,并维护根 CA 的状态(**有效 / 已过期**,见 flows/local-ca.md)。用根 CA 签发出的**内网证书是 certificates 实体**,其列表 / 详情 / 状态 / 续签 / 到期 / 吊销 / 导出全部归 certificates(走证书状态机);本模块作为 certificates "自签"方式的**签发后端**,受其委托执行签发与作废标记,把结果交回,**不复述证书状态**。

全部操作为**本地离线操作**(project §7 离线可用),不依赖外网。MVP 单一角色 operator 对本模块全权(roles.md),无细粒度权限差异。

## 2. 功能列表(按端分组)

> 端点只有一个 `app`(桌面形态 / 服务器形态共享前端);形态差异在功能项内以"仅桌面 / 仅服务器"标注。

### 2.1 端 `app`

**A. 根 CA 管理(使用者直达操作)**

| 功能 | 说明 |
| --- | --- |
| A1 根 CA 查看 | 查看已有根 CA 及其当前状态(有效 / 已过期)、名称 / 标识、有效期(生效 / 失效时间)、创建方式(新建 / 导入)、指纹或序列号;支持查看某根 CA 已签发的内网证书概览(明细归 certificates) |
| A2 创建根 CA | 本地生成密钥对并自签,产出新根 CA;录入名称、有效期等基本信息;成功后进入"有效"(flows/local-ca.md L1) |
| A3 导入根 CA | 导入已有根 CA(提供根 CA 证书 + 配对私钥,私钥受口令保护时提供口令);校验通过后进入"有效",导入证书本身已过期则判为"已过期"(L2) |
| A4 导出根 CA 证书 | 导出根 CA 证书(公开部分)供客户端导入信任库(信任根 CA);仅桌面可保存到本地路径、仅服务器为浏览器下载(形态差异);只读,不改变状态 |

**B. 签发衔接能力(受 certificates 委托,非页面直达操作)**

| 功能 | 说明 |
| --- | --- |
| B1 用根 CA 签发内网证书 | certificates 选"自签"方式发起签发 / 续签时委托本模块:用**有效**根 CA 为目标域名签发内网证书(生成内网证书及私钥并用根 CA 签名、组装证书链),结果交回 certificates;已过期根 CA 拒绝签发(flows/local-ca.md §3.4) |
| B2 内网证书作废标记 | certificates 对"自签"内网证书发起吊销时委托本模块:由签发它的根 CA 记录其作废,结果交回 certificates(§3.5);内网证书吊销态走证书状态机 |

> B1 / B2 是**被委托的后端行为**,由 certificates 触发、经 tasks 调度留痕,不作为 local-ca 页面上的独立操作入口。

> 本模块**不含**:内网证书的列表 / 详情 / 状态管理 / 续签 / 到期 / 导出(全部归 certificates);批量创建 / 批量导出等批量操作;根 CA 自动续期(flows/local-ca.md §4-LC3);把根 CA 自动安装进客户端信任库(导出后由使用者自行导入);多渠道通知(状态供 dashboard)。
> **已裁决项(见 §6,orchestrator 2026-07-16)**:多根 CA 并存 —— 支持并统一管理(已纳入本模块);含私钥的根 CA 备份 / 迁移导出、显式移除根 CA —— MVP 均不提供,后续可增量。

## 3. 页面清单

> 仅列页面名;各页目的 / 功能 / 交互 / 验收在下一层页面 PRD(pages/app/local-ca/…)展开。内网证书的列表 / 详情 / 签发页归 certificates,不在此列。

| 页面 | 一句话定位 |
| --- | --- |

## 4. 数据来源

> 供 architect 作为唯一设计依据,用**业务语言**描述本模块涉及哪些数据及其来源;**不含**表结构 / 字段类型 / 存储机制 / API(由 architect 设计)。敏感数据的具体存储机制由 architect 依 project §7 数据安全约束确定。

| # | 数据 | 来源 | 说明 |
| --- | --- | --- | --- |
| DS1 | 根 CA 条目(本模块核心数据) | **本模块产生并维护** | 每个根 CA 的:名称 / 标识、有效期(生效 / 失效时间)、当前状态(根 CA 状态机:有效 / 已过期)、创建方式(新建 / 导入)、根 CA 证书本体、指纹或序列号、创建时间 |
| DS2 | 根 CA 私钥(敏感数据) | **本模块产生**(创建时生成)/ **接收**(导入时提供) | 用于签发内网证书;属敏感数据(project §7 私钥),安全存储机制由 architect 定;与 DS1 根 CA 配对 |
| DS3 | 根 CA 与其签发内网证书的关联及作废记录 | **本模块维护**(内网证书本体归 certificates) | 记录"某根 CA 签发了哪些内网证书"以及"其中哪些已被标记作废"(支撑 §3.5 吊销衔接);内网证书本体、其状态与有效期归 certificates(其 DS3 记录"经哪个根 CA 签的");作废的具体机制(如作废清单)由 architect 定 |
| DS4 | 签发 / 吊销委托入参 | **消费自** `certificates` | 签发委托:目标域名集合(可含通配符)、指定的根 CA、内网证书有效期口径;吊销委托:待作废内网证书标识、其签发根 CA;本模块据此执行并回传结果,不持有域名 / 证书本体 |
| DS5 | 创建 / 导入录入信息 | **使用者提供** | 创建:根 CA 名称 / 标识、有效期等;导入:已有根 CA 证书、配对私钥、可选解密口令;录入后分别落地为 DS1 / DS2 |

> **敏感边界(待 architect 关注)**:根 CA 私钥(DS2)一旦泄露,凡信任该根 CA 的客户端上其签出的全部内网证书均可被冒用,敏感级别高于单张叶子私钥;存储与导出策略须从严(project §7)。

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 flows/local-ca.md §4 互补(此处记模块 / 功能 / 数据 / 页面相关决策)。

- **DEC1(2026-07-16)· 严守"根 CA 归本模块、内网证书归 certificates"边界**:本模块只做根 CA 的创建 / 导入 / 查看 / 导出 + 受托签发 / 作废标记;内网证书的列表 / 详情 / 状态 / 续签 / 到期 / 吊销 / 导出一律归 certificates。
  - 为什么不把内网证书也放本模块:内网证书与 ACME 证书是同一类实体、共用同一套证书状态机与续签 / 到期 / 吊销逻辑,若在 local-ca 另管一套会与 certificates 重复并制造状态漂移;本模块只当"自签"签发后端。
- **DEC2(2026-07-16)· 页面收敛为 3 页**:根 CA 列表页、根 CA 详情页、新增根 CA 页;创建与导入合于"新增根 CA 页"两种方式,导出 / 查看作为列表 / 详情上的操作。
  - 为什么不拆:创建与导入都是"往列表里加一个根 CA"的同一目的、两种来源,合页更内聚;导出 / 查看依附具体根 CA 上下文,单列会割裂操作与对象(沿 certificates DEC3)。
- **DEC3(2026-07-16)· 支持多个根 CA、统一管理(已裁决)**:模块按"可存在多个根 CA"设计(故有列表页)。
  - 为什么不默认单根 CA:"根 CA 管理混乱 → 统一签发"(project §3)可读作"统一管理多个根 CA",团队分环境(dev / staging)也可能需多根;单根 CA 是多根的子集,后续收敛返工小于反向扩展。**已裁决(orchestrator, 2026-07-16):支持多个根 CA 并存、统一管理,不收敛为单根 CA。**
- **DEC4(2026-07-16)· 导出核心为"根 CA 证书(供客户端信任)",MVP 不开放含私钥导出**:承接 task 明示的导出用途。
  - 为什么不默认含私钥导出:私钥导出属敏感操作(project §7),需单独风险确认,且与"根 CA 跨形态实例迁移"绑定。**已裁决(orchestrator, 2026-07-16):MVP 不开放含私钥 / 迁移导出。代价:根 CA 无法跨形态实例迁移沿用(后续可增量)。**

## 6. 裁决记录(原待澄清项)

> 以下原为需求缺口或跨模块口径,均已由 orchestrator 拍板(2026-07-16),记录如下;后续如变更按增量更新。

1. **多根 CA vs 单根 CA** —— **已裁决(orchestrator, 2026-07-16):支持多个根 CA 并存、统一管理,不收敛为单根 CA**;据此保留列表页 + 详情页 + 新增页三页,certificates DS3"哪个根 CA"表述维持。(不影响根 CA 状态机本身。)
2. **含私钥的根 CA 备份 / 迁移导出** —— **已裁决(orchestrator, 2026-07-16):MVP 不提供含私钥的导出 / 迁移导出**(仅导出公开根 CA 证书)。代价:两形态实例间无法沿用同一根 CA(后续可增量)。
3. **显式移除 / 删除根 CA** —— **已裁决(orchestrator, 2026-07-16):MVP 不提供显式移除根 CA**,生命周期靠"过期后创建 / 导入新根接替"。代价:误建的根 CA 无法清理(后续可增量)。
4. **certificates 依赖复核** —— **已裁决(orchestrator, 2026-07-16):certificates 契约随本批次同步审定,自签委托口径(§3.5 作废机制、签发入参)不变;新增的取消回退转移(T21–T24)不涉及自签委托,local-ca 无需复核。**
