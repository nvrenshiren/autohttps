## 1. 概述

域名管理围绕**"域名"这一核心对象**(glossary),提供域名列表的增删改查、备注与分组、域名与验证方式的关联,以及从域名侧查看其证书覆盖与进入签发的入口。域名是证书签发的**前置对象**:证书必须关联已在本模块存在的域名。

本模块**只管域名对象本身**:证书生命周期(签发 / 续签 / 吊销 / 导出、证书状态机)归 `certificates`,本模块提供域名供其关联并投影其证书态;验证方式的配置与挑战执行归 `acme`,本模块只维护"域名 ↔ 验证方式"的关联;域名与其证书态数据提供给 `dashboard` 呈现。域名**无独立生命周期状态机**(flows/domains.md §2)。

MVP 单一角色 operator 对本模块全权(roles.md),无细粒度权限差异。

## 2. 功能列表(按端分组)

> 端点只有一个 `app`(桌面形态 / 服务器形态共享前端);形态差异在功能项内以"仅桌面 / 仅服务器"标注(本模块暂无形态差异功能)。

### 2.1 端 `app`

**A. 查看**

| 功能 | 说明 |
| --- | --- |
| A1 域名列表查看 | 展示全部域名及其 hostname、是否通配符、分组、备注摘要、**证书态投影**(有效 / 即将到期 / 失败 / 无证书,派生自 certificates)、关联证书数;支持按分组、证书态、hostname 关键字筛选查看 |
| A2 域名详情查看 | 单个域名完整信息:hostname、通配符标识、分组、备注、验证方式关联、**关联证书清单及各自状态**(投影自 certificates,可跳转证书详情) |

**B. 维护**

| 功能 | 说明 |
| --- | --- |
| B1 新增域名 | 录入 hostname(校验格式、自动识别通配符、同实例不重复),可选设置分组 / 备注 / 验证方式关联 |
| B2 编辑域名 | 修改分组 / 备注 / 验证方式关联;hostname 不可编辑(改名 = 删除 + 新增) |
| B3 删除域名 | 二次确认后移除域名对象;若仍被任一证书关联则拦截,提示先在 certificates 处理(flows/domains.md §3.3) |

**C. 关联**

| 功能 | 说明 |
| --- | --- |
| C1 证书关联查看与签发入口 | 从域名侧查看其被哪些证书关联及各状态(投影自 certificates);提供跳转 certificates 为本域名发起签发 / 查看证书详情的入口。签发动作与关联建立归 certificates |
| C2 验证方式关联 | 为域名设置 / 查看其验证方式类别(HTTP-01 / DNS-01)关联;通配符域名限 DNS-01;方式配置与挑战执行归 acme,此处仅建立关联 |

> 本模块**不含**:批量新增 / 删除域名等批量操作、域名维度统计卡片(总数 / 即将到期数归 dashboard)、证书生命周期动作(归 certificates)、验证方式配置与执行(归 acme)、域名启用 / 停用(无独立状态机)。

## 4. 数据来源

> 供 architect 作为唯一设计依据,用**业务语言**描述本模块涉及哪些数据及其来源;**不含**表结构 / 字段类型 / 存储机制 / API(由 architect 设计)。

| # | 数据 | 来源 | 说明 |
| --- | --- | --- | --- |
| DS1 | 域名对象(本模块核心数据) | **本模块产生并维护** | 每个域名的:hostname、是否通配符(由 hostname 判定)、分组、备注;同一实例内 hostname 唯一 |
| DS2 | 域名 ↔ 验证方式关联 | **本模块维护关联**,明细归 `acme` | 该域名倾向使用的验证方式类别(HTTP-01 / DNS-01)的引用;通配符域名固定 DNS-01;方式的具体配置(webroot 路径 / DNS TXT / ACME 账户等)与挑战执行归 acme |
| DS3 | 域名的证书态投影 | `certificates`(只读消费) | 该域名当前被哪些证书关联及各证书状态(证书状态机,引用不复述);"无证书 / 即将到期 / 失败"等呈现均派生自 certificates,本模块不定义证书状态 |
| DS4 | 域名 ↔ 证书关联链 | `certificates`(持有)/ 本模块反向消费 | 证书侧记录其覆盖域名(SAN,含至多一个通配符);本模块从域名侧反查"被哪些证书覆盖";关联在 certificates 签发流程建立 |

> **关联基数(供 architect 关注)**:域名 ↔ 证书为**多对多**——一证多域(SAN),一域可被多证关联;domains 管域名侧,关联链归 certificates(与 modules/certificates.md §4 DS2 / DEC4 一致)。

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 flows/domains.md §5 互补(此处记模块 / 功能 / 数据相关决策)。

- **DECM1(2026-07-16)· 分组为组织 / 筛选属性,不引入批量操作**:分组是域名的可选组织标签,仅用于列表筛选与归类,不作为批量签发 / 批量删除的入口;每个域名归入至多一个分组。
  - 为什么:project §5 明示"分组"但未明示批量场景与分组层级;批量操作为需求未明示功能,与 certificates DEC2 收敛一致;单一分组是"分组"最小可用读法,后续如需多标签再增量。
- **DECM2(2026-07-16)· 域名与验证方式的关联归本模块,明细归 acme**:本模块持有"域名 ↔ 验证方式类别"的选择 / 引用(承接 project §5"验证方式的关联"),验证方式的配置与执行归 acme。
  - 为什么这样切:ACME 验证按域名(identifier)逐个校验,把类别选择置于域名对象便于签发时按域名取用;明细留在 acme 以免边界重叠。**已裁决(orchestrator, 2026-07-16):webroot 路径归 acme(验证执行配置),domains 仅持"域名 ↔ 验证方式类别"关联;acme ↔ domains 边界对齐完成(见 acme DEA5)。**
- **DECM3(2026-07-16)· 页面收敛为 2 页**:域名列表页 + 域名详情页;新增 / 编辑 / 删除作为其上的对话框 / 面板,不单列页面。
  - 为什么不拆:域名对象轻量,增删改是对列表 / 详情上下文的动作;单列页面会割裂操作与对象,与 certificates DEC3 收敛口径一致。
