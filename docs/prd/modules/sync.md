## 1. 概述

备份同步是 autohttps 的**数据保护模块**,把「整座数据目录的一致性快照」经**用户口令整包加密**后,手动推送到用户自有的 **WebDAV** 存储,并能在需要时从远端任一快照**覆盖式恢复**。它回答一个问题:*本机数据(域名 / 证书 / 私钥引用 / ACME 账户 / 根 CA / 任务历史)丢了或要迁机,怎么安全地备份与找回。*

定位与边界:

- **手动触发**,非定时 / 非双向同步 —— 备份与恢复都是 operator 显式动作;不做自动周期备份、不做多机实时同步(见 §5-DEC1)。
- **快照粒度是整座数据目录**,不可只备单证书 / 单域名 —— 备份单元 = 一致性库快照 + 全部密钥材料(见 §5-DEC2)。
- **加密在离开本机前完成**,口令即唯一防线 —— 备份包含全部私钥材料(master.key 在内),用用户口令(age passphrase)整包加密后才上传;WebDAV 服务端 / 任何中间方都无法解密(见 §5-DEC3)。
- **口令不托管** —— WebDAV 登录口令经统一 SecretStore 加密存引用(同 project §7),备份口令**绝不落盘、绝不入库、绝不入日志**,每次备份 / 恢复当次输入(见 §5-DEC4)。

恢复语义:**覆盖式恢复** —— 以备份内容整体替换当前库与密钥材料;恢复前先把当前现场归档(可回滚),完成后**需重启应用**生效(见 §5-DEC5)。

## 2. 功能列表(按端分组)

> 端点只有一个 `app`(桌面形态 / 服务器形态共享前端);本模块无形态专属项。

### 2.1 端 `app`

**A. WebDAV 连接配置**

| 功能 | 说明 |
| --- | --- |
| A1 配置 WebDAV 连接 | 设置服务器地址、**远程目录**(隔离本应用备份与其他项目,默认 `autohttps/`)、用户名、登录口令;口令经 SecretStore 存引用、库内不存本体、读取永不回传 |
| A2 查看连接配置 | 展示服务器地址 / 远程目录 / 用户名 / 口令是否已保存 / 上次备份时间与结果;不回显口令本体 |
| A3 测试连接 | 校验地址可达 + 凭据有效 + 远程目录可写(幂等建目录);结果即时反馈 |
| A4 清除配置 | 删除连接配置连同已存口令密文;远端已有备份不受影响 |

**B. 备份**

| 功能 | 说明 |
| --- | --- |
| B1 立即备份 | 输入备份口令(≥10 位,醒目提示「口令即私钥最后防线」)→ 打一致性库快照(VACUUM INTO)+ 全部密钥材料 → zip → 口令整包加密 → 上传远端目录;文件名带时间戳 |
| B2 查看远端备份列表 | 按备份文件名前缀列出远端快照(名称 / 大小 / 修改时间),新在前 |
| B3 备份结果留痕 | 记录上次备份时间与成功 / 失败结果于配置,供设置页展示 |

**C. 恢复**

| 功能 | 说明 |
| --- | --- |
| C1 从备份恢复 | 选定远端快照 + 输入该备份的口令 → 解密校验 → 恢复前归档当前现场(可回滚)→ 以备份整体覆盖库与密钥材料 → 提示**重启应用**生效 |
| C2 恢复前现场保护 | 恢复前把当前库在线导出归档(`pre-restore.db`),失败可回滚;备份包口令错 / 损坏在写盘前即拦截 |
| C3 恢复后一致性 | 密钥材料全量替换(清孤儿、换 master.key、清身份缓存);库内悬空口令引用对账置空并提示重存 |

> 本模块**不含**:定时 / 自动备份、双向 / 实时同步、单证书或单域名粒度备份、备份口令托管 / 找回、远端备份删除(远端只增,由用户自行在 WebDAV 侧管理)、除 WebDAV 外的其他存储后端(S3 / 网盘等,MVP 不扩展)。

| # | 数据 | 来源 | 说明 |

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 flows/settings.md §5 互补(此处记模块 / 功能 / 数据相关决策)。

- **DEC1(2026-07-19)· 手动快照,不做定时 / 双向同步**:备份与恢复都是 operator 显式动作。
  - 为什么不做自动 / 双向:MVP 最简;自动周期备份涉口令托管(备份需口令,自动则要存口令,放大泄露面)与冲突合并复杂度;双向同步与「单一真相在本机库」冲突。后续确需再增量。
- **DEC2(2026-07-19)· 快照粒度为整座数据目录,不做单证书 / 单域名备份**:备份单元 = 一致性库快照 + 全部密钥材料。
  - 为什么:库与密钥材料相互引用(`*_ref` ↔ 密文),拆开备份会产生悬空引用;整目录快照保证自洽可恢复,实现也最简(VACUUM INTO + 打包 secrets/)。
- **DEC3(2026-07-19)· 离开本机前整包加密,口令即唯一防线**:备份包含全部私钥材料(master.key 在内),用用户口令(age passphrase,≥10 位)整包加密后才上传。
  - 为什么:备份即「私钥全量副本」,强度必须 ≥ 本地静态存储(age 加密);整包加密而非逐文件,保证元数据(manifest)也不外泄;口令强度 = 私钥安全,故 UI 强制最小长度并醒目提示。
- **DEC4(2026-07-19)· 登录口令与备份口令分流,备份口令绝不持久**:WebDAV 登录口令经 SecretStore 存引用(便于免重输);备份口令当次输入、绝不落盘 / 入库 / 入日志。
  - 为什么:登录口令泄露只影响 WebDAV 读写,备份口令泄露 = 全部私钥泄露,风险等级不同;备份口令不托管,丢了无法恢复备份,这是有意的安全取舍。
- **DEC5(2026-07-19)· 恢复为覆盖式 + 现场归档可回滚 + 需重启生效**:以备份整体替换当前库与密钥;恢复前先把当前库在线导出归档,完成后提示重启。
  - 为什么覆盖:增量合并会引入状态机 / 引用一致性复杂度;覆盖语义清晰、可预测。归档与「口令错 / 包损坏在写盘前拦截」保证可回滚;进程持有库连接与密钥缓存,故需重启重连生效。
- **DEC6(2026-07-19)· 备份落独立远程目录,不散在 WebDAV 根目录**:默认 `autohttps/`,可改;拒绝 `..` / 反斜杠防路径穿越。
  - 为什么:同一 WebDAV 账户常被多项目共用,独立目录避免备份混杂与误删;归一与校验在服务端做,前端不兜底。
