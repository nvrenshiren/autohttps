## 1. 证书与域名

| 中文名     | 英文 / 缩写         | 定义(业务口径)                                                        |
| ---------- | ------------------- | ----------------------------------------------------------------------- |
| 域名       | Domain              | 需要 HTTPS 保护的主机名,如 `example.com`;autohttps 管理的核心对象      |
| 通配符域名 | Wildcard            | 形如 `*.example.com` 的域名,覆盖其全部一级子域;只能经 DNS-01 验证签发   |
| 证书       | Certificate         | 证明某域名身份、用于建立 HTTPS 的数字凭证,有明确有效期                  |
| 私钥       | Private Key         | 与证书配对的密钥,属敏感数据;泄露即证书失去意义(project §7 数据安全)   |
| 证书链     | Certificate Chain   | 从叶子证书到根 CA 的信任链条;部署时通常需一并提供                        |
| 有效期     | Validity Period     | 证书的生效到失效时间区间                                                 |
| 到期       | Expiry              | 证书达到失效时间;到期后浏览器不再信任                                   |
| 即将到期   | Expiring Soon       | 距到期不足"续签策略提前天数"的状态,是自动续签与红点提醒的触发条件       |
| 吊销       | Revoke              | 主动声明某证书作废(如私钥泄露),使其提前失效                           |

## 2. ACME 自动签发

| 中文名      | 英文 / 缩写      | 定义(业务口径)                                                             |
| ----------- | ---------------- | ---------------------------------------------------------------------------- |
| ACME        | ACME             | 自动证书管理协议;autohttps 借此向 CA 自动申请 / 续签证书                     |
| 证书颁发机构 | CA               | 签发受浏览器信任证书的机构,如 Let's Encrypt                                 |
| ACME 账户   | ACME Account     | 在某 CA 上的身份(通常绑定邮箱),申请证书的前提                              |
| 挑战 / 验证 | Challenge        | CA 要求证明"你确实控制该域名"的验证过程;autohttps 支持 HTTP-01 与 DNS-01     |
| HTTP-01     | HTTP-01(webroot) | 在域名的 web 根目录放置指定验证文件,CA 通过 HTTP 访问校验                    |
| webroot     | webroot          | web 服务器对外提供静态文件的根目录;HTTP-01 验证文件放于此                    |
| DNS-01      | DNS-01(手动)     | 在域名 DNS 解析中添加指定 TXT 记录供 CA 校验;MVP 为**手动**添加(project §6.2) |

## 3. 自签根 CA

| 中文名     | 英文 / 缩写       | 定义(业务口径)                                                        |
| ---------- | ----------------- | ----------------------------------------------------------------------- |
| 自签根 CA  | Local Root CA     | 由用户自己创建、非公共信任的根证书;用于内网 / 本地开发场景统一签发       |
| 内网证书   | Internal Cert     | 由自签根 CA 签发的证书;仅在信任该根 CA 的客户端上有效                    |
| 信任根 CA  | Trust Root CA     | 把根 CA 导入客户端信任库,使其签发的内网证书被认可的动作                 |

## 4. 续签、任务与运行

| 中文名     | 英文 / 缩写     | 定义(业务口径)                                                            |
| ---------- | --------------- | -------------------------------------------------------------------------- |
| 续签       | Renewal         | 在证书到期前重新签发以延续有效期                                            |
| 自动续签   | Auto-Renewal    | 按续签策略在"即将到期"时自动触发续签,无需人工                              |
| 续签策略   | Renewal Policy  | 全局设置:提前多少天续签、是否自动(project §5 settings)                    |
| 续签失败   | Renewal Failed  | 续签任务执行未成功的状态(如验证失败 / 网络问题);需红点提醒并可重试        |
| 任务       | Task            | 一次签发 / 续签的执行单元,含时间、结果、日志;历史留痕(project §5 tasks)  |
| 运行形态   | Run Mode        | 桌面形态(Tauri 窗口 + 托盘)或服务器形态(守护进程 + Web UI)(project §4)  |
| 守护进程   | Daemon          | 服务器形态下 7×24 常驻的后台进程                                            |
| 系统托盘   | Tray            | 桌面形态常驻于系统托盘的入口,关窗不退出                                    |
