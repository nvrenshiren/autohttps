## 1. 目的

签发证书页用于**为域名发起首次签发**:选定一个或多个已存在域名(SAN,至多一个通配符)、选择签发方式(公共 ACME / 自签根 CA)、触发签发。它是证书生命周期的**入口动作**;续签 / 重新签发不在本页(归证书详情页)。

验证方式(HTTP-01 / DNS-01)的**向导明细归 acme**,本页仅提供入口 / 衔接,不展开向导。

---

## 2. 功能清单

| # | 功能 | 说明(业务口径) |
| --- | --- | --- |
| F1 选择域名 | 从已在域名管理存在的域名中选择一个或多个组成本次签发范围(SAN);至多包含一个通配符域名 |
| F2 选择签发方式 | 二选一:公共 ACME 或 自签根 CA |
| F3 验证方式衔接 | 选公共 ACME 时,衔接 acme 验证方式;通配符域名限 DNS-01;向导明细在 acme 展开,本页仅提供入口 |
| F4 触发首次签发 | 确认后创建证书条目并触发首次签发,证书进入「待签发」(T1) |

> 本页**不含**:续签 / 重新签发(归证书详情页);验证方式的具体配置与挑战执行(webroot 路径 / DNS TXT,归 acme);自签根 CA 的创建 / 导入(归 local-ca);批量签发(DEC2)。

## 3. 页面流转

| 触发 | 去向 / 结果 | 说明 |
| --- | --- | --- |
| 触发签发 | 创建证书条目 → 证书进入「待签发」→ 通常跳转该证书详情页 / 证书列表页看进度 | 后续执行进度与失败原因在任务 / 详情查看 |

## 4. 交互说明

- **选择域名**:只能从**已在域名管理存在**的域名中选(证书必须关联已存在域名);可多选组成 SAN;至多一个通配符域名(modules/certificates.md DEC4)。从域名详情页进入时**预置该域名**。
- **选择签发方式**:公共 ACME 与 自签根 CA 二选一。
  - 公共 ACME:签发经 acme 验证域名控制权后向 CA 取证;需衔接验证方式(见下)。
  - 自签根 CA:由 local-ca 用根 CA 直接签发,不经 ACME 域名验证(flows/certificates.md §3.1)。
- **验证方式衔接(仅公共 ACME)**:通配符域名只能经 DNS-01 验证签发(glossary);签发方式 / 验证方式选择受此约束。验证方式向导(HTTP-01 webroot / DNS-01 手动 TXT)的**明细归 acme**,本页仅提供入口 / 衔接,不在本页展开配置。
- **触发签发**:确认后系统创建证书条目并登记签发任务,证书进入「待签发」;后续「签发中 / 有效 / 签发失败」的推进与失败原因在任务 / 证书详情查看,不在本页。
- **仅首次签发**:本页只用于首次签发;对已有证书的续签 / 重新签发在证书详情页发起。

---

## 5. 验收要点

> 业务口径,每条可判真伪。

1. 可从**已在域名管理存在**的域名中选择一个或多个组成本次签发范围(SAN)。
2. 不能选择未在域名管理中存在的域名。
3. 一次签发至多包含一个通配符域名。
4. 可选择签发方式:公共 ACME 或 自签根 CA(二选一)。
5. 从域名详情页进入签发时,预置该域名。
6. 选择公共 ACME 且含通配符域名时,验证方式限定为 DNS-01。
7. 本页仅提供验证方式向导的入口 / 衔接,不在本页展开 HTTP-01 / DNS-01 的明细配置。
8. 触发签发后创建证书条目,证书进入「待签发」状态。
9. 触发签发后可进入该证书详情 / 证书列表查看后续进度。
10. 本页仅用于首次签发;不提供对已有证书的续签 / 重新签发。
11. 本页不提供批量签发。
