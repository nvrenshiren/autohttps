## 1. 目的

域名详情页呈现**单个域名的完整信息**,并作为其**编辑**(分组 / 备注 / 验证方式关联)、**关联证书查看**与**签发入口**的落点。hostname 是域名身份**不可编辑**(改名 = 删除 + 新增)。删除域名以「是否被证书关联」**硬拦截**。

---

## 2. 功能清单

| # | 功能 | 说明(业务口径) |
| --- | --- | --- |
| F1 完整信息展示 | hostname、通配符标识、分组、备注、验证方式关联、关联证书清单及各证书状态 |
| F2 编辑域名 | 修改分组 / 备注 / 验证方式关联;**hostname 不可编辑** |
| F3 验证方式关联 | 设置 / 查看验证方式类别(HTTP-01 / DNS-01);通配符域名限 DNS-01;明细归 acme |
| F4 关联证书查看 | 查看覆盖本域名的证书清单及各状态(投影自 certificates),可逐条跳转证书详情 |
| F5 发起签发入口 | 跳转签发证书页,为本域名发起签发(预置本域名) |
| F6 删除域名 | 二次确认后移除;若仍被任一证书关联则**硬拦截** |

> 本页**不含**:hostname 编辑(改名 = 删 + 增,DECD2);证书生命周期动作(归 certificates,本页仅提供查看与签发入口);验证方式的具体配置与挑战执行(webroot 路径 / DNS TXT,归 acme)。

## 4. 交互说明

- **完整信息展示**:hostname、是否通配符、分组、备注、验证方式关联,以及**关联证书清单及各证书状态**(状态投影自 certificates 证书状态机,引用不复述)。
- **编辑**:可修改分组 / 备注 / 验证方式关联;**hostname 只读不可改**——需改名时以「删除旧域名 + 新增新域名」替代(DECD2),避免已关联证书的 SAN 与域名对象错位。
- **验证方式关联**:设置 / 查看验证方式类别(HTTP-01 / DNS-01);**通配符域名只能关联 DNS-01**(glossary);此处仅建立「域名 ↔ 验证方式类别」关联,具体配置(webroot 路径 / DNS TXT)与挑战执行**委托 acme**,本页提供衔接入口不展开明细。
- **关联证书查看**:列出覆盖本域名的证书及各状态,可逐条跳转证书详情;签发动作与「域名 ↔ 证书关联」的建立均在 certificates 完成,本页只查看与提供入口。
- **发起签发**:提供「为本域名发起签发」入口,跳转签发证书页并预置本域名。
- **删除域名**:二次确认;若该域名仍被**任一现存证书**关联则**拦截删除**并提示先在证书管理处理相关证书(DECD3);未被任何证书关联时移除该域名并返回域名列表页。本页不代为删改证书。

---

## 5. 验收要点

> 业务口径,每条可判真伪。

1. 展示单域名完整信息:hostname、是否通配符、分组、备注、验证方式关联、关联证书清单及各证书状态。
2. hostname 不可编辑。
3. 可编辑分组 / 备注 / 验证方式关联并保存。
4. 通配符域名的验证方式关联限定为 DNS-01。
5. 验证方式的具体配置(webroot 路径 / DNS TXT)不在本页展开,本页仅建立验证方式类别关联(明细归 acme)。
6. 关联证书清单中每条证书可跳转至其证书详情页。
7. 提供为本域名发起签发的入口,进入签发证书页且预置本域名。
8. 删除域名需二次确认。
9. 被任一现存证书关联时删除被拦截,提示先在证书管理处理相关证书。
10. 未被任何证书关联时删除成功并返回域名列表页。
11. 本页不提供 hostname 编辑;不提供证书生命周期动作(续签 / 吊销 / 删除证书等)。
