## 1. 概述

系统设置是 autohttps 的**全局配置聚合模块**,承载四类一次设定、长期生效的配置:**运行形态配置**(桌面开机自启 / 服务器监听地址与端口)、**数据存储路径**、**全局续签策略**(续签提前天数 / 自动续签开关)、**默认 ACME 账户**。

本模块是**续签策略的唯一定义点**:certificates 消费本模块的提前天数与自动续签开关来判定"即将到期"与是否自动续签(见 flows/certificates.md,引用不复述),本模块不判定证书状态、不执行续签。其余配置各被相应模块消费(见 flows/settings.md §4 影响范围)。

MVP **不做鉴权**(project §9-D4 / roles §4),故本模块**无账户 / 登录 / 权限**类设置。单一角色 operator 对本模块全权(roles.md),可查看并修改各配置项;其中**数据存储路径与当前运行形态为只读展示**(见 §3)。运行形态由部署载体既定,设置内不提供切换。

## 2. 功能列表(按端分组)

> 端点只有一个 `app`(桌面形态 / 服务器形态共享前端);形态差异在功能项内以"仅桌面 / 仅服务器"标注。

### 2.1 端 `app`

**A. 运行形态配置**

| 功能 | 说明 |
| --- | --- |
| A1 查看当前运行形态 | 只读展示本实例为桌面形态或服务器形态;据此决定形态相关项的可见性(仅桌面 / 仅服务器);不提供切换形态 |
| A2 开机自启开关 | **仅桌面**:设置桌面应用是否随系统登录自动启动并常驻托盘 |
| A3 监听地址 / 端口配置 | **仅服务器**:设置守护进程 Web UI 的监听地址与端口;设为对外可达地址时须**明确提示公网暴露风险**(roles §3);默认仅监听本机 / 可信内网 |

**B. 数据存储路径**

| 功能 | 说明 |
| --- | --- |
| B1 查看数据存储路径(只读) | 只读展示本地持久化数据(域名 / 证书 / 私钥 / ACME 账户密钥 / 根 CA 材料 / 任务历史)的落地根路径;路径在部署 / 首次启动时确定,运行期不可更改(见 §5-DEC3) |

**C. 全局续签策略**

| 功能 | 说明 |
| --- | --- |
| C1 配置续签提前天数 | 设置"距到期多少天判定为即将到期"的阈值;供 certificates 判定"即将到期"、驱动自动续签与红点(策略在此定义,判定归 certificates,引用不复述) |
| C2 自动续签开关 | 设置是否在"即将到期"时自动发起续签;关闭则仅提醒、靠手动续签 |

**D. 默认 ACME 账户**

| 功能 | 说明 |
| --- | --- |
| D1 选择默认 ACME 账户 | 从 acme 已配置账户中选一个作为签发 / 续签的默认账户;候选来源 acme,本模块只存"默认指向";无账户或默认被删时,签发发起处引导去 acme 添加 / 显式选择 |

> 本模块**不含**:账户 / 登录 / 权限设置(project §9-D4);通知渠道配置(MVP 仅 dashboard 红点,project §6.2);主题 / 语言 / 日志级别 / 更新检查等未明示项;续签重试次数 / 间隔配置与扫描周期配置(见 flows/settings.md §5-SF2/SF3);运行形态切换与形态间数据迁移(project §4)。

| # | 数据 | 来源 | 说明 |

| DS1 | 运行形态配置 | **本模块产生并维护** | 开机自启开关(仅桌面);监听地址与监听端口(仅服务器)。均为全局单份 |

## 5. 决策记录(append-only)

> 只增不改;记"决定了什么 / 为什么不做另一选项"。与 flows/settings.md §5 互补(此处记模块 / 功能 / 数据相关决策)。

- **DEC1(2026-07-16)· 设置聚合为单页分区,不按配置域拆多页**:运行形态 / 存储 / 续签策略 / ACME 默认账户四区同页呈现。
  - 为什么不拆:MVP 配置项少,单页分区信息密度合适;拆多页徒增导航与心智成本。后续项增多再评估拆分。
- **DEC2(2026-07-16)· 无鉴权 / 账户 / 权限设置项**:承接 project §9-D4 与 roles §4。
  - 为什么不做:MVP 不做登录与多用户权限,访问权=网络可达性;故设置内无账户体系可配。
- **DEC3(2026-07-16)· 存储路径运行期只读展示、不可更改(已裁决 orchestrator 2026-07-16)**:路径在部署 / 首次启动时确定,运行期只读展示、不可更改,无迁移语义。
  - 为什么:MVP 最简,避免运行期迁移的复杂度与失败风险;后续确需再增量(同 flows/settings.md §5-SF5)。
- **DEC4(2026-07-16)· 默认 ACME 账户仅存"默认指向",账户本体归 acme**:本模块不复制账户明细,只保存默认账户引用。
  - 为什么:尊重模块边界(project §5 acme 负责账户配置),避免与 acme 账户管理重复定义、产生漂移。
